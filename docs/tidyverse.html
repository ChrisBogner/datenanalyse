6.4 Die Grammatik der Datenmanipulation ‚Äì dplyr

Die Funktion count() geh√∂rt zum Paket dplyr, das f√ºr Datentransformationen zust√§ndig ist. Es ist mal wieder eine Grammatik. Dieses Paket enth√§lt 5 Grundfunktionen (alle nach Verben benannt, damit man gleich wei√ü, was frau tut üòÑ):

Funktion Bedeutung
filter() W√§hle Daten anhand ihrer Werte
arrange() Sortiere Zeilen
select() W√§hle Variablen anhand ihrer Namen
mutate() Erstelle neue Variablen als Funktionen vorhandener Variablen

Wir m√∂chten nur von einer bestimmten Station die Anzahl der Messwerte wissen m√∂chten, dann filtern wir vorher.

  filter(STATIONS_ID == '2667') %>%
  count()
## # A tibble: 1 x 1
##       n
##   <int>
## 1 13200

Beim Filtern l√§uft eine logische Abfrage. D.h. es wird bei jeden Eintrag in STATION_ID nachgesehen, ob da der Wert 2667 steht. Wenn da 2667 steht, dann gibt == ein TRUE zur√ºck, wenn da etwas anderes steht, dann gibt == ein FALSE zur√ºck. Und die Funktion filter() beh√§lt nur die Zeilen, bei denen == ein TRUE zur√ºck gegeben hat.

Weiter wichtige logische und relationale Operatoren finden Sie hier in der Hilfe zu filter(). Hier ein paar einfache Beispiele

Operator Bedeutung
==/ > / >= ist die linke Seite gleich / gr√∂√üer / gr√∂√üer-gleich als die rechte Seite
!= ist die linke Seite ungleich der rechten Seite
